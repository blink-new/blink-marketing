## What This Skill Covers

- **Platform:** X (Twitter) and LinkedIn
- **Post type:** Feature launches and product announcements
- **Voice:** Kai's founder personal account OR Blink brand account
- **Formats:** Short punchy post, thread (X), short/medium/long (LinkedIn)

---

## The Single Most Important Pattern

Every high-performing launch post in the reference examples uses a **reply-to-claim giveaway**. This is not optional — it is the primary engagement mechanism that drives reach on both platforms.

**The mechanic:**
- Announce the launch
- Make a bold, specific claim about what it does
- Offer something free (credits, a month of access, a course)
- Require a reply/comment/RT to claim — this triggers the algorithm
- On LinkedIn: also require a connection so you can DM (grows follower count)

Without this mechanic, a launch post is just an announcement. With it, it's a reach engine.

## Voice: Kai (Founder Personal) vs. Blink (Brand Account)

### Kai's Voice
- First person. Direct. Founder energy.
- Can share personal context, vision, or stakes ("we built this", "I'll DM you")
- More human — can express excitement without it feeling like marketing
- LinkedIn: can go long with feature details + personal conviction
- X: should be short, punchy, confident — not a press release

### Blink Brand Voice
- Third person or "we"
- More product-forward, less personal story
- Cleaner and more polished than Kai's voice
- Same structural rules apply — result first, bold claim, giveaway mechanic
- Avoid: sounding like a corporate announcement

## X (Twitter) Formats

### Format 1: The One-Thing Post (Highest ceiling for virality)

Structure:
```
[Introducing / Launching / dropping] [Product/Feature] — [one-line bold claim].

[One sentence of proof or specificity.]

[Optional: one more line that deepens the claim.]

Reply [word] / Comment below [to claim free X].
```

Rules:
- Max 4 lines of content before the CTA
- The bold claim must be the sharpest, most specific version of the value prop
- No feature lists. One thing only.
- The giveaway CTA goes last and is a single line

**Best for:** Major launches, new product announcements, anything that has a clear "wow" demo.

---

### Format 2: Short Feature List Post (3–6 bullets)

Structure:
```
[Introducing / Today we're launching] [Feature] for [Blink/platform].

[One sentence on what it enables.]

[Feature bullets — max 5, use → or numbered list]
→ [Feature]
→ [Feature]
→ [Feature]

[Giveaway CTA — reply/comment to claim]
```

Rules:
- Each bullet must be a capability, not a description
- Lead bullet should be the most impressive one
- Keep bullets parallel — all nouns, or all verb phrases
- Still keep it tight — don't pad

**Best for:** Feature drops with multiple capabilities (like Blink Agent Builder), where the breadth of what's included IS the story.

---

### Format 3: Thread (For complex launches or storytelling)

Use a thread when:
- The launch requires context to land (why we built this, what problem it solves)
- You want to dominate the feed for a sustained period
- The story is as important as the announcement

Thread structure:
```
Tweet 1 (Hook): The announcement + boldest claim. Must work standalone.
Tweet 2: The "why" — what problem this solves, who it's for
Tweet 3–6: Feature/capability deep-dives, one per tweet
Tweet 7: Social proof — numbers, beta results, who's already using it
Tweet 8: The giveaway CTA + reply mechanic
```

Rules:
- Tweet 1 must work as a standalone post — not everyone will read the thread
- Each tweet should have its own point — no filler tweets that just bridge
- Giveaway CTA goes in the final tweet, not tweet 1

**Best for:** Milestone launches, fundraise announcements paired with a feature, anything where the backstory adds weight.

## LinkedIn Formats

### Short Post (Under 300 chars) — Orchids model
Use when: The product/feature speaks for itself. No explanation needed.

Structure:
```
[Introducing / Launching] [Product/Feature] — [one-line bold claim].

[One sentence of proof or specificity. Optional.]

Comment below to get [free X].
```

### Medium Post (300–800 chars) — Orchids + giveaway model
Use when: The feature needs one paragraph of context, but you want it readable in full without expanding.

Structure:
```
[Introducing / Launching] [Product/Feature] — [bold claim].

[1–2 sentences of what it does and why it matters.]

[Optional: 3–5 feature bullets with →]

Comment below [or Repost + comment] to get [free X].
[If Kai personal: "(Make sure to connect with me so I can DM you)"]
```

Rules:
- Keep it above the fold — no "see more" truncation if possible
- If using bullets, max 5
- The connection request line (Kai only) goes in parentheses, last

### Long Post (800+ chars) — Blink Agent Builder model
Use when: The launch is major, the feature list is genuinely impressive, and depth signals credibility.

Structure:
```
[Introducing / Launching] [Product/Feature] — [bold positioning line].

[1–2 sentences of what it enables.]

[3–5 sentences of core value / how it works]

[Feature list with →, 5–10 items]

[1–2 sentences of unique technical depth or differentiation]

[Closing vision line — what this makes possible]

Repost and comment below — I'll DM you [free X].
(Make sure to connect with me so I can DM you)
```

Rules:
- Opening line must be the headline — the rest supports it, not the other way around
- Feature list uses → arrows, not bullets
- Close with vision, not features — end on aspiration
- The DM mechanic requires the connection note on LinkedIn (Kai personal only)
- Brand account: "Comment below for [free X]" — no DM mechanic

## The Giveaway Mechanic — Execution Details

This is the highest-leverage element of any launch post. Get it right.

**What to give away:**
- Free credits (50K, 100K — specific numbers work better than "free trial")
- Free month of access
- Bonus content (course, playbook) paired with credits
- Default: 1 free month or 50K credits for a launch post

**The CTA line options (X):**
- "Reply to claim"
- "Comment below to get [X] free credits"
- "RT and comment '[word]' — [free thing] on us"
- "Reply '[word]' and we'll DM you [free thing]"

**The CTA line options (LinkedIn):**
- "Comment below to get [free thing]"
- "Repost and comment below — I'll DM you [free thing]"
- Always add on Kai's account: "(Make sure to connect with me so I can DM you)"

**What makes it work algorithmically:**
- Comments trigger reach more than likes
- Reposts trigger reach more than comments
- The specific word mechanic ("comment 'BLINK'") makes it easy to track and signals intent
- The connection requirement on LinkedIn builds Kai's network while distributing the offer

## Writing Process

1. **Identify the one thing.** What is the single most impressive or surprising fact about this launch? That becomes your opening claim. If you can't identify one thing, the launch positioning isn't ready yet.

2. **Write the opening line first.** It must work standalone. If someone reads only this line, do they know something remarkable just happened? Test: would you stop scrolling for this?

3. **Choose the format** based on the launch size and complexity:
   - One clear wow moment → Format 1 (One-Thing)
   - Multiple strong capabilities → Format 2 (Feature List) or Long LinkedIn
   - Complex story or major milestone → Thread or Long LinkedIn

4. **Write the giveaway CTA last.** It should feel like a natural conclusion — "we're excited, here's how to get involved" — not a bolted-on marketing add.

5. **Read it aloud.** If any sentence sounds like a press release, rewrite it.

---

## Quality Checks

Before finalizing any post:
- [ ] Opening line works as a standalone tweet/post — could go viral on its own
- [ ] Contains one specific, bold claim (dollar amount, speed, named competitor, or quantified result)
- [ ] Giveaway mechanic is present and the CTA is a single, clean line
- [ ] No corporate language: "leverage", "seamless", "disruptive", "game-changing", "unlock"
- [ ] Brand name is correct: Blink.new or Blink (never BLINK, blink, blinkdotnew)
- [ ] For Kai's LinkedIn: connection note included if DM mechanic is used
- [ ] Post is not trying to say more than one thing (especially on X)

---

## What to Avoid

**On X:**
- Long feature lists — they kill momentum
- Opening with context before the claim ("As many of you know, building apps is hard...")
- Hashtags — they look like 2018 marketing
- Tagging other accounts in the main post (use replies/quote tweets for that)

**On LinkedIn:**
- Emoji overload
- Paragraph-length sentences
- Burying the giveaway in the middle of the post
- Ending on a feature instead of a vision statement or CTA

**On both:**
- Announcing "we're excited to announce" — just announce it
- Passive voice
- Vague claims ("a better way to build apps") without specifics
- Making it feel like an ad — it should feel like news
